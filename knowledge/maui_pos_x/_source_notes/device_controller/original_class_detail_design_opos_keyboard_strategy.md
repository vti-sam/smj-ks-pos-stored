## 概要

Windows環境でNamedPipe経由でMauiPOSHostと連携し、専用キーイベントを受信するストラテジクラス。
キーボードデバイスのライフサイクル管理（接続・切断）およびキー入力待機ループによるコールバック方式のキーイベント通知を提供する。

### メソッド名: キー入力待機

#### 戻り値

| 型   | 概要 |
| ---- | ---- |
| Task | -    |

#### 引数

| 修飾子 | 型                              | 引数名（論理）     | 引数名（物理） | 概要                             |
| ------ | ------------------------------- | ------------------ | -------------- | -------------------------------- |
| -      | Action\<DedicatedKeyEventArgs\> | キー受信コールバック | onKeyReceived  | キーデータ受信時のコールバック   |

#### 発生例外

| 型                         | 概要                                         |
| -------------------------- | -------------------------------------------- |
| OperationCanceledException | キャンセルトークンによりループが中断された場合 |

#### 処理概要

| 項番 | 処理概要                                                                                  |
| ---- | ----------------------------------------------------------------------------------------- |
| 1    | 停止中フラグとキャンセルトークンを確認しながらループを実行する。                          |
| 1.1  | 名前付きパイプでキー待機コマンドを送信し、レスポンスを受信する。                         |
| 1.2  | レスポンスが空または "ERROR" で始まる場合、500ms 待機して再試行する。                    |
| 1.3  | レスポンス解析でDedicatedKeyEventArgsを生成する。                                        |
| 1.4  | 解析成功の場合、コールバックを呼び出す。                                                |
| 2    | キャンセル操作（`OperationCanceledException`等）により停止要求があった場合、キー入力待機ループを終了する。                |
| 3    | パイプ通信障害等の予期しない例外（その他の例外）が発生した場合、1000ms 待機して再試行プロセス（ループ）を継続する。       |

| ------ | ------ | ------------------ | -------------- | -------------------- |
| -      | string | レスポンス文字列   | response       | パイプレスポンス     |

#### 発生例外

| 型  | 概要 |
| --- | ---- |
| -   | -    |

#### 処理概要

| 項番 | 処理概要                                                                             |
| ---- | ------------------------------------------------------------------------------------ |
| 1    | レスポンスをセミコロンで3分割する。                                                  |
| 2    | 分割数が3未満または先頭が "0" でない場合、null を返却する。                          |
| 3    | 2番目の要素を整数変換し keyValue、3番目を label として DedicatedKeyEventArgs を生成する。 |
| 4    | 整数変換に失敗した場合、null を返却する。                                            |
