## 3. クラス定義

### 3.1 全体方針

ViewModelの全プロパティを自動保存せず、`[PersistSnapshot]`が付与されたプロパティだけを保存対象とする。
Presentation層はDBへ直接アクセスせず、`IViewModelSnapshotService`を経由して保存・復元する。

SQLiteへの永続化はInfrastructure層に閉じ込める。
ViewModel側は`SnapshotViewModelBase`を継承することで、画面表示・非表示およびアプリ停止・復帰時の保存復元処理を共通化する。

## 6. 処理フロー

### 6.1 保存フロー

```mermaid
sequenceDiagram
    autonumber
    participant App as App / Page lifecycle
    participant VM as SnapshotViewModelBase
    participant Port as IViewModelSnapshotService
    participant Session as ILocalStateSessionService
    participant DB as SQLite

    App->>VM: OnDisappearingAsync / OnStoppingAsync / Window.Deactivated
    VM->>VM: [PersistSnapshot] property抽出
    VM->>VM: 500ms以内に保存済みの場合はskip
    VM->>Port: SaveSnapshotAsync(sessionKey, snapshotKey, properties, route)
    Port->>Session: GetOrCreateActiveSessionAsync(sessionKey)
    Session->>DB: local_state_sessions upsert
    Port->>DB: view_model_snapshots upsert
    Port->>Session: UpdateLastRouteAsync(sessionKey, route)
```

### 6.2 復元フロー

```mermaid
sequenceDiagram
    autonumber
    participant App as App / Page lifecycle
    participant VM as SnapshotViewModelBase
    participant Port as IViewModelSnapshotService
    participant DB as SQLite

    App->>VM: OnAppearingAsync / OnResumingAsync / Window.Activated
    VM->>Port: GetSnapshotAsync(sessionKey, snapshotKey)
    alt cache exists
        Port-->>VM: memory cache
    else cache missing
        Port->>DB: payload_json取得
        DB-->>Port: snapshot row
        Port-->>VM: Dictionary
    end
    VM->>VM: JsonElementをproperty型へ変換して反映
```

## 7. 注意事項

- パスワード、token、PIN、secure credential、hardware handleは保存しない。
- master dataやAPIから再取得可能な大容量データは保存しない。
- 本仕様はViewModelの入力途中状態を復元するための基盤であり、売上・決済などの業務データ永続化DBではない。
- `local_settings` tableおよびPreferences連携は本scope外。
- `session_key`を業務flowごとに分けることで、複数flowの並行保持に対応する。
- `PageDisappearing`、`Window.Deactivated`、`Window.Stopped`が短時間に連続発火する可能性があるため、snapshot保存は`SemaphoreSlim`で並列実行を防止し、500ms以内の重複保存をskipする。
