# Estimate Schedule Customer Wording

- Customer-facing estimate artifact should be treated as `見積仕様書・工程表`, not as detailed WBS.
- Do not include internal notes such as repository paths, source-code progress, approval steps, or wording like loss/discount.
- Main focus for May-July 2026 is documentation, testcase, verification, review, and finalization. Code-related items are supporting technical scope, not the document's primary emphasis.
- Grouping for the estimate: Host / Device Connection Server including CAFISArch payment terminal integration, Local State / SQLite, Monitoring / Sentry, Testcase / Verification, Review / Coordination.
- Effort convention selected for this estimate: calculated effort `5.25人月`, presented effort `3.00人月`, VTI adjustment `2.25人月`.
- Monthly calculated effort: May `1.50人月`, June `1.75人月`, July `2.00人月`.
- Customer-facing Japanese wording for the adjustment should use soft phrasing such as `VTI側で調整する形としております`, not red/discount/loss terminology.
